```ts
export interface LocaleConfig {
  // language name
  lang?: string;
  // HTML title
  title?: string;
  // HTML description
  description?: string;
  // Display text in corresponding language
  label: string;
  // Navbar config, of the same type as `themeConfig.nav`
  nav?: NavItem[];
  // Sidebar config, of the same type as `themeConfig.sidebar`
  sidebar?: Sidebar;
  // Outline title
  outlineTitle?: string;
  // The text of the last updated time
  lastUpdatedText?: string;
  // The text of the edit link
  editLink?: EditLink;
  // The text of the prev page
  prevPageText?: string;
  // The text of the next page
  nextPageText?: string;
}
```

As you see, many fields in `themeConfig` can also be configured in `LocaleConfig`, including:

- [nav](/api/config-theme#nav)
- [sidebar](/api/config-theme#sidebar)
- [editLink](/api/config-theme#editlink)
- [lastUpdatedText](/api/config-theme#lastupdatedtext)
- [outlineTitle](/api/config-theme#outlinetitle)
- [prevPageText](/api/config-theme#prevpagetext)
- [nextPageText](/api/config-theme#nextpagetext)

But the `LocaleConfig` has higher priority. It overrides the corresponding field in `themeConfig`.
